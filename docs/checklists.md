# Чеклісти для тестування

## Модуль: Управління довідником предметів та академічних груп

### Функціональність
- [ ] Відкривається сторінка списку предметів
- [ ] Відкривається сторінка списку академічних груп
- [ ] Користувач може відкрити форму створення нового предмета
- [ ] Користувач може створити предмет з валідною унікальною назвою
- [ ] Після успішного створення предмет з’являється у списку
- [ ] Користувач може відкрити форму редагування предмета
- [ ] Користувач може змінити назву предмета на нову валідну
- [ ] Після редагування в списку відображаються оновлені дані
- [ ] Користувач може створити академічну групу з валідними даними
- [ ] Після створення група з’являється у списку
- [ ] Користувач може редагувати дані академічної групи
- [ ] Користувач може видалити предмет, який не має зв’язків
- [ ] Користувач може видалити групу, яка не має зв’язків

### Негативні сценарії
- [ ] Неможливо створити предмет з порожньою назвою
- [ ] Неможливо створити предмет з назвою, що складається лише з пробілів
- [ ] Неможливо створити предмет з дублюючою назвою
- [ ] Неможливо зберегти назву предмета, що перевищує допустиму довжину
- [ ] Неможливо створити групу з порожньою назвою
- [ ] Неможливо створити групу з дублюючою назвою
- [ ] Неможливо видалити предмет, якщо він має зв’язки з іншими сутностями
- [ ] Неможливо видалити групу, якщо вона має зв’язки з іншими сутностями
- [ ] При невалідному вводі користувач бачить повідомлення про помилку
- [ ] При помилці валідації дані у формі не зникають без потреби

### UI/UX
- [ ] Усі кнопки CRUD-подій видимі та зрозуміло підписані
- [ ] Обов’язкові поля візуально позначені
- [ ] Повідомлення про помилки зрозумілі користувачу
- [ ] Після успішного створення/редагування відображається повідомлення про успіх
- [ ] Діалог підтвердження видалення зрозумілий і містить назву сутності
- [ ] Інтерфейс коректно відображається без зламаної верстки
